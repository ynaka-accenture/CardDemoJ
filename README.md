## 概要

日本版CardDemo(JCARDDEMO)は、AWS CardDemo を**日本の金融機関の業務システム**
として再構築した、メインフレーム クレジットカード管理アプリケーションである。
IBM EBCDIC日本語(IBM漢字 / DBCS-Host, CCSID 930/939/5035)を文字コードモデルとし、漢字氏名・漢字住所・
漢字摘要を扱う。データ項目名・コメント・帳票・画面をすべて日本語化し、
**国産メインフレーム資産のモダナイゼーション(特にUTF-8移行)の困難性**を
忠実に再現する検証用テストベッドである。

本家 CardDemo がシングルバイト(ASCII)のみを扱うのに対し、日本版は
**多バイト文字(DBCS)・SO/SIシフトコード・外字・ベンダー固有文字コード境界**
という、日本市場のレガシー移行で必ず直面する壁を実コードとして含む。

## 説明

日本版CardDemo は、以下のモダナイゼーション検証を目的とする日本仕様の
メインフレームアプリケーションである。

- アプリケーション資産の発見・分析(日本語フィールドを含む)
- COBOL→Java 変換(MAJALIS等)における日本語データの等価性検証
- 文字コード移行(EBCDIC日本語→UTF-8)の困難性の体験
- オンライン(CICS BMS)・バッチ・IMS-DB2・MQ連携の総合的な移行評価

日本の金融機関で実際に稼働しているシステムを模し、口座・カード・顧客・取引・
与信・明細の各業務を、漢字を含む現実的なデータで処理する。

## 技術構成

| 区分 | 技術 |
|------|------|
| 言語 | IBM Enterprise COBOL for z/OS(DBCS利用者語で日本語データ名に対応) |
| オンライン | CICS + BMS(DBCS/MIXED属性による漢字画面) |
| データベース | VSAM(KSDS)、IMS-DB(階層型)、DB2(関係型) |
| メッセージング | MQ(CCSID変換を伴う漢字メッセージ) |
| 文字コード | IBM EBCDIC日本語(IBM漢字/DBCS-Host, CCSID 930/939/5035)、SO(0x0E)/SI(0x0F)混在 |
| バッチ | JCL によるジョブ制御 |

## オプション機能

本家同様、3つのオプションアプリケーションを日本語化して含む。

| オプション | 内容 | 日本語化の焦点 |
|-----------|------|---------------|
| VSAM-MQ | 口座照会のMQ連携 | MQ MD の CCSID変換境界での漢字文字化け |
| 与信(IMS-DB2-MQ) | 与信明細のIMS-DB2格納 | IMS DBD/DB2 CCSID の文字コード定義 |
| 取引タイプ(DB2) | 取引タイプ説明のDB2管理 | DB2列のバイト幅 vs 文字数の桁あふれ |

## 文字コードモデル

日本版CardDemo は **EBCDIC日本語(JEFモデル)** を原資産の文字コードとする。

```
混在フィールド(漢字+英数字)の構造:
  [SO]漢字漢字[SI] A B C
  0E  XXXXXXXX 0F  41 42 43

  SO(0x0E) = Shift-Out(以降2バイトDBCS)
  SI(0x0F) = Shift-In(以降1バイトSBCS)
  漢字1文字 = 2バイト(EBCDIC) → UTF-8では3バイト
```

詳細は `doc/文字コード設計書.md` を参照。

## 導入と実行

GnuCOBOL + Python3 環境で稼働確認できる。

```bash
bash scripts/run_batch.sh   # 英字名版バッチのビルド&実行(GnuCOBOL検証)
```

日本語データ項目名(`顧客番号`・`漢字姓` 等)は IBM Enterprise COBOL for z/OS の
**DBCS 利用者語(DBCS user-defined words)** 機能を用いる。これは z/OS 上の
Enterprise COBOL が正式にサポートする機能で、EBCDIC ソース中で SO/SI に
囲まれた日本語をデータ名・段落名に使用できる。ローカル検証に用いる GnuCOBOL は
DBCS 利用者語に非対応のため、実行確認用には英字名版を併載している。

## アプリケーション詳細

### 利用者機能
- サインオン(利用者認証)

### 管理者機能
- 利用者管理
- 取引タイプ管理(漢字説明)

### プログラム一覧
`doc/プログラム一覧.md`(全47プログラムの日本語名・役割・トラップ対応)を参照。

DBCS/MIXED属性(`ATTRB=(...,DBCS)` / `SOSI=YES`)で定義する。

## 日本語化トラップ

UTF-8移行を阻む11種のトラップを意図的に埋め込んでいる。
詳細は `doc/トラップ一覧.md`。

| # | トラップ | 概要 |
|---|---------|------|
| T1 | 固定バイト幅 | X(n)=nバイト固定。漢字UTF-8 3バイトで枠超過 |
| T2 | SO/SIシフト | DBCS区間の0x0E/0x0F |
| T3 | PIC N/NATIONAL | 各国語項目の処理系依存 |
| T4 | 外字 | Unicode正準マッピング無し |
| T5 | 波ダッシュ | U+301C vs U+FF5E |
| T6 | 円記号0x5C | ¥ vs \ |
| T7 | 半角カナ | JIS X 0201 |
| T8 | DBCS空白 | 0x4040/0x8140 |
| T9 | SO/SI不整合 | 実務データ破損 |
| T10 | 全角数字 | 半角との混在 |
| T11 | 整列順 | EBCDIC vs Unicode |
| +α | MQ/DB2/IMS CCSID境界 | ミドルウェア境界の文字コード変換 |

## モダナイゼーション検証

本テストベッドは、COBOL→Java変換ツール(MAJALIS等)・動的等価性テスト・
Phase2意味再構築の検証対象として、日本市場に即したベンチマークを提供する。
本家CardDemoでは見えない「データのバイト表現(層3)」の壁を、実際の
業務シナリオで体験できる。

### 日本語データ名版について

`app/cbl/顧客帳票.cbl`・`app/cbl/漢字解析.cbl` および `app/cpy/` 配下の
日本語名コピー句(顧客レコード.cpy 等)は、データ項目名・段落名・コメントを
すべて日本語化した**本番相当ソース**である。IBM Enterprise COBOL for z/OS の
DBCS 利用者語(DBCS user-defined words)機能に対応する。GnuCOBOLは多バイトデータ名に非対応のため、
実行検証は英字名版(CBCUSJ1C 等)で行う(`scripts/run_batch.sh`)。
