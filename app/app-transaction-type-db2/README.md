# 取引タイプ(DB2)オプション(日本版CardDemo)

取引タイプ説明を DB2 で管理するオプションアプリケーションの日本語版。

## 日本語化の焦点: DB2桁あふれ

取引タイプの漢字説明を扱う。DB2の CHAR(n)/VARCHAR(n) の n が
EBCDICバイト前提(漢字2バイト)で定義されているため、UTF-8(漢字3バイト)
への移行で桁あふれ・切詰めが発生する。

## プログラム
- COBTUPJT — 取引タイプ更新バッチ(英字名・実行可能、DB2桁あふれ検出)
- COTRTJLI — 取引タイプ一覧
- COTRTJUP — 取引タイプ更新(オンライン)
